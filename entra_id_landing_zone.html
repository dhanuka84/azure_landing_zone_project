Microsoft Entra ID + Azure Landing Zone

A mental model that connects the Identity Plane (Entra ID) with the Landing Zone’s Control and Resource planes, showing how RBAC, Policy, and Network controls align to create a secretless, policy-driven platform.

Identity Plane

Authenticates humans and workloads. “Never trust, always verify.”

  • Human Identities
    Admins & operators are PIM-enabled; access conditioned by Conditional Access & MFA.
    PIM (JIT) Conditional Access
  • Workload Identities (UAMI)
    User-Assigned Managed Identities for apps (AKS, App Service, Functions) — no secrets.
    Key Vault Secrets User AcrPull Storage Blob Data Reader
  • Federated Identities (OIDC)
    Azure DevOps / GitHub Actions exchange signed OIDC tokens for Entra tokens. No client secrets.
    Secretless CI/CD Short-lived tokens

Control Plane

Governs at scale. “Enforce least privilege & compliance.”

  • Management Groups
    Platform → Non-Prod → Prod. Policy & RBAC inheritance down to subscriptions.
  • Azure RBAC
    Grant roles to Entra groups, UAMI, and OIDC identities at MG/Subscription/RG/Resource scope.
  • Azure Policy
    Deny public IPs; enforce tags; private link required; DeployIfNotExists for Private DNS links.
    Deny Modify DeployIfNotExists

Resource Plane

Where workloads live: subscriptions, VNets, and services.

  • Subscriptions
    Connectivity (Hub), Management, Dev, QA, Prod.
  • Hub & Spoke VNets
    Central egress via Firewall; Private Endpoints in spokes; Private DNS linked.
  • Services
    Key Vault, ACR, AKS, Storage, SQL — public network disabled; PE required.

Zero Trust Chain

  1. Authenticate (Entra ID)
  2. Authorize (RBAC + Policy)
  3. Isolate (Firewall + NSGs + Private Endpoints/DNS)
  4. Observe (Defender for Cloud + Logs)

Network Boundary

Azure Firewall

Default route (0.0.0.0/0) from spokes via UDR; threat intelligence & egress control.

NSGs

L3/L4 micro-segmentation per subnet; deny-by-default with minimal allows.

Private Link & Private DNS

PaaS traffic stays private; DNS linked across subscriptions with Policy remediation.
